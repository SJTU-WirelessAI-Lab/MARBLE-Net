# MARBLE-Net (WCNC Channel & Positioning)

Official code release for the paper:

**MARBLE-Net: Learning to Localize in Multipath Environment with Adaptive Rainbow Beams**  
arXiv: https://arxiv.org/abs/2511.06971

---

## Overview

This repository provides:
- **Ray-tracing channel dataset generation** with **Sionna** (TensorFlow) via `generate_h5.py`.
- **Training & evaluation** of position-estimation methods in **PyTorch**, including:
  - k-NN baseline: `train_knn_gpu.py`
  - RaiNet baseline: `train_RaiNet.py`
  - Proposed model (MARBLE-Net): `train_MARBLE.py`
- Shared helpers in `wcnc_utils.py`.

---

## Repository Structure

- `generate_h5.py`  
  Generate an HDF5 dataset of wideband channel observations using Sionna ray-tracing.

- `train_knn_gpu.py`  
  k-NN positioning baseline (GPU-accelerated neighbor search).

- `train_RaiNet.py`  
  RaiNet baseline positioning model.

- `train_MARBLE.py`  
  MARBLE-Net training and evaluation.

- `wcnc_utils.py`  
  Common utilities (data loading, preprocessing, metrics, logging, etc.).

- `requirements.txt`  
  Python dependencies.

- `LICENSE`  
  MIT License.

## Installation

### Option A: Conda (recommended)

```bash
conda create -n marble python=3.10 -y
conda activate marble
pip install -r requirements.txt
```

### Notes
Dataset generation requires TensorFlow + Sionna (ensure your TF/CUDA versions are compatible).

Training scripts are implemented in PyTorch (GPU recommended).

## Quick Start
1) Generate the HDF5 channel dataset (Sionna + TensorFlow)
```bash
python generate_h5.py
```
This script generates the ray-tracing channel dataset in HDF5 format.
Please refer to `generate_h5.py` for the output directory, dataset schema, and scene configuration.

2) Train / Evaluate Models (PyTorch)

**k-NN baseline**
```bash
python train_knn_gpu.py --data_root sionna_1109/los --k_neighbors 5 --gpu_id 0
```

**RaiNet baseline**
```bash
python train_RaiNet.py --data_root sionna_1109/los --training_mode iteration --gpu_id 0
```

**MARBLE-Net (proposed)**
```bash
python train_MARBLE.py --data_root sionna_1109/los --gpu_id 0
```

Show all options
```bash
python train_knn_gpu.py -h
python train_RaiNet.py -h
python train_MARBLE.py -h
```

## Data
The dataset is stored in HDF5 format generated by Sionna ray-tracing.

Each sample contains channel-derived observations and the corresponding position label.

Train/val/test splits are handled by the training scripts (see script arguments and code).

## Reproducibility Tips
Fix random seeds (NumPy / PyTorch) for deterministic runs if needed.

Log the dataset path, training mode, and hyperparameters for each experiment.
